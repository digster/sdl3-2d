2D Track

Twelve recipes that take you from opening a window to shipping a playable Pong clone, all in C++ with SDL3's callback model and the high-level SDL_Renderer drawing API. Each one builds on the previous — work through them in order.

Curriculum

  • 01 Hello Window
    The four SDL_App* callbacks, opening a window, clearing the screen.
     Beginner
  • 02 Drawing Primitives
    Set-then-draw colour model, rectangles, lines, points, the gfx:: helpers.
     Beginner
  • 03 Handling Input
    Keyboard and mouse events inside SDL_AppEvent; quitting cleanly.
     Beginner
  • 04 Delta Time & Movement
    Frame-rate-independent motion with SDL_GetTicksNS.
     Beginner
  • 05 Loading Textures
    Surfaces, textures, BMP loading, source/destination rects.
     Intermediate
  • 06 Sprite Animation
    Sprite sheets, frame timers, source-rect cycling.
     Intermediate
  • 07 Collision Detection
    AABB intersection, point-in-rect, circle-circle distance checks.
     Intermediate
  • 08 Audio Basics
    Loading a WAV with SDL_LoadWAV, audio streams, one-shot effects.
     Intermediate
  • 09 Text Rendering
    A dependency-free bitmap-font approach — draw characters from a sprite sheet.
     Intermediate
  • 10 Tilemap Rendering
    A 2D grid + tileset, scrolling camera, viewport-culled drawing.
     Advanced
  • 11 Particle Systems
    A pool of particles with lifetime, velocity, fade; spawn-update-cull.
     Advanced
  • 12 Mini Game: Pong
    Capstone — paddles, ball, scoring, simple AI. Everything from 01–11 in one file.
     Advanced

How to run the code

Every recipe's Complete listing is a drop-in replacement for examples/cpp/callbacks.cpp. Replace that file's contents, then build with make run-cpp from the repo root. The recipe will compile against the existing gfx.hpp / gfx.cpp in that folder.

New to the build setup? See the root README.
