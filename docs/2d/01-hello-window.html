Recipe 01 — Hello Window

The smallest legal SDL3 callback program: open a window, paint it a colour, and quit cleanly.

What you'll learn

  • The four SDL_App* callbacks SDL3 wants you to define.
  • How to put your app state in a heap struct and let SDL thread it back to you.
  • The "clear → draw → present" lifecycle every 2D frame follows.
  • Why SDL3 returns true on success (and not 0).

Prerequisites

SDL3 installed (brew install sdl3 on macOS) and the repo's examples building (make run-cpp succeeds). No prior recipe needed — this is recipe one.

Walkthrough

SDL3 supports two loop styles. The classic style is the while (running) game loop in your own main(). The other style — the one we'll use everywhere in these recipes — is the callback model: you define four functions, and SDL calls them at the right moments. This model survives the trip to web (Emscripten) and mobile platforms unchanged, because on those platforms your program isn't allowed to own the main loop.

1. Enable the callback model

Two lines, at the very top of the file, before any other SDL header: 

#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>   // provides the real entry point (also on macOS)

The #define tells <SDL3/SDL_main.h> to wire up a main() for us that calls our four callbacks. Without it you'd be back to writing your own loop. Including SDL_main.h is required on macOS even in callback mode, because Cocoa needs SDL's startup shim.

2. Declare your state

We won't use globals. Instead, every long-lived thing the app cares about — window handles, timers, game state — lives in one struct on the heap. SDL holds a pointer to it for us and passes it back into every callback. For now, that's just the window and the renderer: 

struct AppState {
    SDL_Window   *window   = nullptr;
    SDL_Renderer *renderer = nullptr;
};

3. SDL_AppInit — one-time setup

SDL calls this once at startup. We init the SDL subsystem we need (SDL_INIT_VIDEO), create a window with a renderer attached, and store the resulting AppState* in *appstate. That pointer is what SDL will thread back into the other three callbacks. 

SDL_AppResult SDL_AppInit(void **appstate, int argc, char *argv[])
{
    (void)argc; (void)argv;

    // Optional metadata used by the window manager / about box.
    SDL_SetAppMetadata("Hello Window", "1.0", "com.example.sdl3.hello");

    // SDL3 convention: true == success. (SDL2 returned 0 — be careful.)
    if (!SDL_Init(SDL_INIT_VIDEO)) {
        SDL_Log("SDL_Init failed: %s", SDL_GetError());
        return SDL_APP_FAILURE;
    }

    AppState *st = new AppState();

    if (!SDL_CreateWindowAndRenderer("Hello Window", 800, 600,
                                     SDL_WINDOW_RESIZABLE,
                                     &st->window, &st->renderer)) {
        SDL_Log("SDL_CreateWindowAndRenderer failed: %s", SDL_GetError());
        delete st;
        return SDL_APP_FAILURE;
    }
    SDL_SetRenderVSync(st->renderer, 1);   // pace to the display refresh rate

    *appstate = st;            // SDL hands this back into every callback
    return SDL_APP_CONTINUE;   // tell SDL we're ready to run
}

Every callback returns an SDL_AppResult: SDL_APP_CONTINUE means "keep running", SDL_APP_SUCCESS means "exit cleanly", and SDL_APP_FAILURE means "exit with an error".

4. SDL_AppEvent — one event at a time

SDL calls this once for every incoming event. The minimal job is to notice when the user closed the window and quit: 

SDL_AppResult SDL_AppEvent(void *appstate, SDL_Event *e)
{
    (void)appstate;
    if (e->type == SDL_EVENT_QUIT) {
        return SDL_APP_SUCCESS;   // window close button or system-requested quit
    }
    return SDL_APP_CONTINUE;
}

5. SDL_AppIterate — one frame

SDL calls this once per frame. The 2D drawing lifecycle is always clear → draw → present. Right now we have nothing to draw, so it's just clear-and-present: 

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    // Pick the clear colour (R, G, B, A), then issue the clear.
    // "Set then draw" is the universal pattern for SDL_Renderer.
    SDL_SetRenderDrawColor(st->renderer, 18, 18, 28, 255);
    SDL_RenderClear(st->renderer);

    // (Future recipes will draw something between these two calls.)

    SDL_RenderPresent(st->renderer);   // swap the back buffer to the screen
    return SDL_APP_CONTINUE;
}

6. SDL_AppQuit — clean up

SDL calls this once at shutdown. Destroy what we created, in reverse order of creation. Do not call SDL_Quit() yourself — the callback runtime does that for you after this function returns. 

void SDL_AppQuit(void *appstate, SDL_AppResult result)
{
    (void)result;
    AppState *st = static_cast<AppState *>(appstate);
    if (st) {
        SDL_DestroyRenderer(st->renderer);
        SDL_DestroyWindow(st->window);
        delete st;
    }
}
Why a heap object? SDL stores a single void* and threads it back into every callback. A heap-allocated AppState gives that pointer something stable to point at for the lifetime of the program — and skipping globals means the file scales cleanly when state grows.

Complete listing

Drop this into examples/cpp/callbacks.cpp (replacing its contents) and run make run-cpp. 

/*
 * 01-hello-window.cpp — the smallest legal SDL3 callback program.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>

struct AppState {
    SDL_Window   *window   = nullptr;
    SDL_Renderer *renderer = nullptr;
};

SDL_AppResult SDL_AppInit(void **appstate, int argc, char *argv[])
{
    (void)argc; (void)argv;

    SDL_SetAppMetadata("Hello Window", "1.0", "com.example.sdl3.hello");

    if (!SDL_Init(SDL_INIT_VIDEO)) {
        SDL_Log("SDL_Init failed: %s", SDL_GetError());
        return SDL_APP_FAILURE;
    }

    AppState *st = new AppState();

    if (!SDL_CreateWindowAndRenderer("Hello Window", 800, 600,
                                     SDL_WINDOW_RESIZABLE,
                                     &st->window, &st->renderer)) {
        SDL_Log("SDL_CreateWindowAndRenderer failed: %s", SDL_GetError());
        delete st;
        return SDL_APP_FAILURE;
    }
    SDL_SetRenderVSync(st->renderer, 1);

    *appstate = st;
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *appstate, SDL_Event *e)
{
    (void)appstate;
    if (e->type == SDL_EVENT_QUIT) {
        return SDL_APP_SUCCESS;
    }
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);
    SDL_SetRenderDrawColor(st->renderer, 18, 18, 28, 255);
    SDL_RenderClear(st->renderer);
    SDL_RenderPresent(st->renderer);
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult result)
{
    (void)result;
    AppState *st = static_cast<AppState *>(appstate);
    if (st) {
        SDL_DestroyRenderer(st->renderer);
        SDL_DestroyWindow(st->window);
        delete st;
    }
}

Try it yourself

  1. Change the clear colour to something garish — e.g. SDL_SetRenderDrawColor(st->renderer, 220, 50, 200, 255). Convince yourself SDL_AppIterate really is running every frame.
  2. Remove the SDL_WINDOW_RESIZABLE flag and observe that you can no longer drag the corner.
  3. Change the window's initial size to 1280×720. Notice how nothing else in the file needs to change — the renderer follows the window.
  4. Log "frame!" inside SDL_AppIterate and confirm it prints at ~60 Hz on a 60 Hz display thanks to vsync.
