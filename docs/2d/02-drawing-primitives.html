Recipe 02 — Drawing Primitives

Points, lines, rectangles, circles. The "set colour, then draw" pattern every SDL3 primitive obeys.

What you'll learn

  • The set-then-draw model used by every SDL_Render* call.
  • How to draw points, lines, outlined and filled rectangles in plain SDL3.
  • How the repo's gfx:: namespace ships outlined / filled circle and triangle helpers on top.
  • Why everything in SDL3's 2D API takes float coordinates.

Prerequisites

Recipe 01 — Hello Window. You have a window opening and clearing every frame.

Walkthrough

Every drawing call in SDL3's 2D API reads the renderer's current draw colour. So the universal pattern is: 

SDL_SetRenderDrawColor(renderer, r, g, b, a);   // pick a colour (0..255)
SDL_RenderRect(renderer, &some_rect);          // draw with it

That's two SDL calls per shape. You can change the colour between every draw — the renderer doesn't care.

Primitives shipped with SDL3

Out of the box, SDL_Renderer can draw: points (SDL_RenderPoint, single pixel), lines (SDL_RenderLine), rectangle outlines (SDL_RenderRect), and filled rectangles (SDL_RenderFillRect). All take float coordinates (SDL3 dropped the integer-only SDL2 API).

What this repo adds: gfx::

Circles and triangles aren't shipped as built-ins, so this repo's examples/cpp/gfx.hpp adds them. It also wraps the built-ins so every shape lives behind the same call style: 

gfx::set_color (ren, r, g, b, a);              // pick (alpha included)
gfx::point     (ren, x, y);
gfx::line      (ren, x1, y1, x2, y2);
gfx::rect      (ren, x, y, w, h);              // outline
gfx::fill_rect (ren, x, y, w, h);              // filled
gfx::circle    (ren, cx, cy, radius);          // outline
gfx::fill_circle(ren, cx, cy, radius);         // filled
gfx::triangle     (ren, x1,y1, x2,y2, x3,y3);  // outline
gfx::fill_triangle(ren, x1,y1, x2,y2, x3,y3);  // filled

Use whichever you like in the recipes. We'll use gfx:: for compactness; if you want to see what it expands to, read its .cpp.

Putting it together

Replace the body of SDL_AppIterate from recipe 01 with a sequence of pick a colour, draw something pairs: 

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    gfx::clear(st->renderer, 18, 18, 28);            // dark slate background

    // Outlined rectangle, top-left.
    gfx::set_color(st->renderer, 255, 200, 60, 255);
    gfx::rect(st->renderer, 40, 40, 160, 100);

    // Filled rectangle, beside it.
    gfx::set_color(st->renderer, 60, 200, 120, 255);
    gfx::fill_rect(st->renderer, 220, 40, 160, 100);

    // A diagonal line.
    gfx::set_color(st->renderer, 80, 200, 255, 255);
    gfx::line(st->renderer, 40, 180, 380, 260);

    // Outlined and filled circles.
    gfx::set_color(st->renderer, 255, 120, 200, 255);
    gfx::circle(st->renderer, 130, 380, 55);
    gfx::set_color(st->renderer, 120, 160, 255, 255);
    gfx::fill_circle(st->renderer, 320, 380, 55);

    // A pair of triangles.
    gfx::set_color(st->renderer, 255, 240, 120, 255);
    gfx::triangle(st->renderer, 470, 440, 550, 300, 630, 440);
    gfx::set_color(st->renderer, 255, 110, 90, 255);
    gfx::fill_triangle(st->renderer, 660, 440, 720, 300, 780, 440);

    SDL_RenderPresent(st->renderer);
    return SDL_APP_CONTINUE;
}
Why float? SDL3 chose float coordinates so the rasterizer can place geometry sub-pixel-accurately. If you pass whole-number floats you get the same look as integer SDL2; if you pass 0.5 offsets you can avoid the "off by one row" alignment bugs that plagued SDL2 line drawing.

Complete listing

Drop into examples/cpp/callbacks.cpp and make run-cpp.

/*
 * 02-drawing-primitives.cpp — every gfx:: primitive in one window.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>
#include "gfx.hpp"

struct AppState {
    SDL_Window   *window   = nullptr;
    SDL_Renderer *renderer = nullptr;
};

SDL_AppResult SDL_AppInit(void **appstate, int, char *[])
{
    SDL_SetAppMetadata("Drawing Primitives", "1.0", "com.example.sdl3.primitives");

    if (!SDL_Init(SDL_INIT_VIDEO)) {
        SDL_Log("SDL_Init failed: %s", SDL_GetError());
        return SDL_APP_FAILURE;
    }

    AppState *st = new AppState();

    if (!SDL_CreateWindowAndRenderer("Drawing Primitives", 800, 600,
                                     SDL_WINDOW_RESIZABLE,
                                     &st->window, &st->renderer)) {
        SDL_Log("CreateWindowAndRenderer failed: %s", SDL_GetError());
        delete st;
        return SDL_APP_FAILURE;
    }
    SDL_SetRenderVSync(st->renderer, 1);

    *appstate = st;
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *, SDL_Event *e)
{
    if (e->type == SDL_EVENT_QUIT) return SDL_APP_SUCCESS;
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    gfx::clear(st->renderer, 18, 18, 28);

    gfx::set_color(st->renderer, 255, 200, 60, 255);
    gfx::rect(st->renderer, 40, 40, 160, 100);

    gfx::set_color(st->renderer, 60, 200, 120, 255);
    gfx::fill_rect(st->renderer, 220, 40, 160, 100);

    gfx::set_color(st->renderer, 80, 200, 255, 255);
    gfx::line(st->renderer, 40, 180, 380, 260);

    gfx::set_color(st->renderer, 255, 120, 200, 255);
    gfx::circle(st->renderer, 130, 380, 55);

    gfx::set_color(st->renderer, 120, 160, 255, 255);
    gfx::fill_circle(st->renderer, 320, 380, 55);

    gfx::set_color(st->renderer, 255, 240, 120, 255);
    gfx::triangle(st->renderer, 470, 440, 550, 300, 630, 440);

    gfx::set_color(st->renderer, 255, 110, 90, 255);
    gfx::fill_triangle(st->renderer, 660, 440, 720, 300, 780, 440);

    SDL_RenderPresent(st->renderer);
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (st) {
        SDL_DestroyRenderer(st->renderer);
        SDL_DestroyWindow(st->window);
        delete st;
    }
}

Try it yourself

  1. Replace every gfx:: call with the raw SDL_Render* equivalent for one shape, to convince yourself gfx:: is paper-thin.
  2. Pass 128 as the alpha to gfx::set_color for the filled rectangle and watch it blend over the line below. (SDL3 renderers default to SDL_BLENDMODE_BLEND, so alpha "just works.")
  3. Draw a grid: nested for loops drawing gfx::point at every (10*i, 10*j) — a quick way to feel how SDL_AppIterate runs once per frame regardless of how much work it does.
