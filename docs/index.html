Why callback-style C++?

SDL3 introduced an alternative to the classic while (running) game loop: define four functions — SDL_AppInit, SDL_AppEvent, SDL_AppIterate, SDL_AppQuit — and SDL drives them for you. This is the recommended model in SDL3 because the same code stays portable to platforms (web via Emscripten, iOS) where your program is forbidden from owning the main loop.

The recipes below use this style consistently. Per-app state lives in a single heap AppState struct whose pointer SDL threads back into every callback — no globals, no static state. If you'd rather see a traditional hand-written loop for comparison, the repo's examples/c/traditional.c shows it in plain C.

New here? Start with the README for installation (brew install sdl3 on macOS) and how to build the existing examples with make run-cpp or make run-gpu. Recipe 01 of each track gives you a minimal working file you can drop straight into one of those folders.

How each recipe is laid out

  • What you'll learn — the concepts the recipe introduces.
  • Prerequisites — earlier recipes you should be comfortable with.
  • Walkthrough — narrative tutorial, code in small pieces, with the why spelled out.
  • Complete listing — the full file, ready to paste into examples/cpp/ or examples/cpp-gpu/ and run.
  • Try it yourself — small experiments to lock in the concept before moving on.

Code blocks are colourised by Prism.js via a CDN. If you're offline before the CDN caches, the code still reads — it's just monochrome.
