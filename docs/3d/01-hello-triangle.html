Recipe 01 — Hello Triangle

The smallest legal SDL_GPU program. A device, two shaders, a pipeline, and one rasterised triangle.

What you'll learn

  • What SDL_GPU is and how it differs from SDL_Renderer.
  • The four-step setup: device → window → shaders → pipeline.
  • The per-frame command sequence: acquire command buffer → acquire swapchain → render pass → submit.
  • The [[vertex_id]] trick — drawing without any vertex buffer at all.
  • Why a NULL swapchain texture is normal, not an error.

Prerequisites

SDL3 installed and make run-gpu already working with the repo's existing triangle. Comfortable with the 2D-track callback model (2D recipe 01) — this recipe reuses the same SDL_App* skeleton.

Walkthrough

Why SDL_GPU exists

SDL_Renderer is high-level: you ask for a rectangle, SDL figures out how. SDL_GPU is a tier lower: you describe a graphics pipeline explicitly — shaders, vertex layout, blend state — and SDL maps that description to Metal (macOS), Vulkan (Linux), or D3D12 (Windows). The two are mutually exclusive on the same window.

The repo's existing examples/cpp-gpu/triangle.cpp is exactly this recipe — read it side by side with the listing below.

The two shaders

A graphics pipeline always has at least two stages: a vertex shader (runs per vertex, outputs a clip-space position) and a fragment shader (runs per pixel, outputs a colour). We'll embed them as MSL source strings — the Metal backend compiles them at runtime, so no offline tools are needed. 

static const char *kVS = R"msl(
#include <metal_stdlib>
using namespace metal;

struct VSOut {
    float4 position [[position]];   // required: clip-space
    float4 color;                   // smuggled into the fragment stage
};

vertex VSOut vertexMain(uint vid [[vertex_id]])
{
    // Three corners baked into the shader. No vertex buffer needed.
    const float2 P[3] = { float2( 0.0,  0.5),
                          float2(-0.5, -0.5),
                          float2( 0.5, -0.5) };
    const float3 C[3] = { float3(1,0,0), float3(0,1,0), float3(0,0,1) };

    VSOut o;
    o.position = float4(P[vid], 0.0, 1.0);
    o.color    = float4(C[vid], 1.0);
    return o;
}
)msl";

static const char *kFS = R"msl(
#include <metal_stdlib>
using namespace metal;

struct VSOut { float4 position [[position]]; float4 color; };

fragment float4 fragmentMain(VSOut in [[stage_in]]) { return in.color; }
)msl";

The rasterizer interpolates the per-vertex color across the triangle's face, so a pixel halfway between the red and green corners arrives at the fragment shader already yellow.

Init: device, window, shaders, pipeline

// 1. GPU device — we promise to supply MSL shaders.
st->device = SDL_CreateGPUDevice(SDL_GPU_SHADERFORMAT_MSL,
                                 /*debug=*/true, /*name=*/nullptr);

// 2. A plain window (no SDL_Renderer here).
st->window = SDL_CreateWindow("Hello Triangle", 800, 600, SDL_WINDOW_RESIZABLE);

// 3. Bind window to device — creates the swapchain.
SDL_ClaimWindowForGPUDevice(st->device, st->window);

// 4. Compile the two shaders.
SDL_GPUShaderCreateInfo vsi{};
vsi.code       = (const Uint8 *)kVS;
vsi.code_size  = std::strlen(kVS);
vsi.entrypoint = "vertexMain";
vsi.format     = SDL_GPU_SHADERFORMAT_MSL;
vsi.stage      = SDL_GPU_SHADERSTAGE_VERTEX;
SDL_GPUShader *vs = SDL_CreateGPUShader(st->device, &vsi);

SDL_GPUShaderCreateInfo fsi{};
fsi.code       = (const Uint8 *)kFS;
fsi.code_size  = std::strlen(kFS);
fsi.entrypoint = "fragmentMain";
fsi.format     = SDL_GPU_SHADERFORMAT_MSL;
fsi.stage      = SDL_GPU_SHADERSTAGE_FRAGMENT;
SDL_GPUShader *fs = SDL_CreateGPUShader(st->device, &fsi);

// 5. Build the pipeline — the immutable "how to draw" description.
SDL_GPUColorTargetDescription ct{};
ct.format = SDL_GetGPUSwapchainTextureFormat(st->device, st->window);

SDL_GPUGraphicsPipelineCreateInfo pci{};
pci.vertex_shader   = vs;
pci.fragment_shader = fs;
pci.primitive_type  = SDL_GPU_PRIMITIVETYPE_TRIANGLELIST;
pci.target_info.color_target_descriptions = &ct;
pci.target_info.num_color_targets         = 1;

st->pipeline = SDL_CreateGPUGraphicsPipeline(st->device, &pci);

// Pipeline owns its compiled stages; we can release the standalone shaders now.
SDL_ReleaseGPUShader(st->device, vs);
SDL_ReleaseGPUShader(st->device, fs);

Per-frame: command buffer → render pass → submit

SDL_GPUCommandBuffer *cmd = SDL_AcquireGPUCommandBuffer(st->device);

SDL_GPUTexture *swap = nullptr;
Uint32 sw = 0, sh = 0;
SDL_WaitAndAcquireGPUSwapchainTexture(cmd, st->window, &swap, &sw, &sh);

if (swap) {                              // NULL is normal (minimized / no display)
    SDL_GPUColorTargetInfo cti{};
    cti.texture     = swap;
    cti.clear_color = SDL_FColor{ 0.07f, 0.07f, 0.10f, 1.0f };
    cti.load_op     = SDL_GPU_LOADOP_CLEAR;
    cti.store_op    = SDL_GPU_STOREOP_STORE;

    SDL_GPURenderPass *pass = SDL_BeginGPURenderPass(cmd, &cti, 1, nullptr);
    SDL_BindGPUGraphicsPipeline(pass, st->pipeline);
    SDL_DrawGPUPrimitives(pass, /*num_vertices=*/3, /*num_instances=*/1, 0, 0);
    SDL_EndGPURenderPass(pass);
}

SDL_SubmitGPUCommandBuffer(cmd);          // also presents the frame (if non-NULL swap)
NULL swapchain is normal. SDL_WaitAndAcquireGPUSwapchainTexture can return true (success) yet hand back a NULL texture — the window is occluded/minimized, or there's no display at all (CI). Skip drawing, still submit the empty command buffer, carry on.

Complete listing

This is essentially the repo's existing triangle.cpp — open it for the most thoroughly commented version. The listing below is the same code with the comments trimmed for compactness. 

/*
 * 3d/01-hello-triangle.cpp — minimal SDL_GPU pipeline that rasterises one
 * RGB-cornered triangle. See examples/cpp-gpu/triangle.cpp for the
 * full-commented version.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>

#include <cstring>

static const char *kVS = R"msl(
#include <metal_stdlib>
using namespace metal;
struct VSOut { float4 position [[position]]; float4 color; };
vertex VSOut vertexMain(uint vid [[vertex_id]]) {
    const float2 P[3] = { float2(0.0, 0.5), float2(-0.5,-0.5), float2(0.5,-0.5) };
    const float3 C[3] = { float3(1,0,0), float3(0,1,0), float3(0,0,1) };
    VSOut o;
    o.position = float4(P[vid], 0.0, 1.0);
    o.color    = float4(C[vid], 1.0);
    return o;
}
)msl";

static const char *kFS = R"msl(
#include <metal_stdlib>
using namespace metal;
struct VSOut { float4 position [[position]]; float4 color; };
fragment float4 fragmentMain(VSOut in [[stage_in]]) { return in.color; }
)msl";

struct AppState {
    SDL_Window              *window   = nullptr;
    SDL_GPUDevice           *device   = nullptr;
    SDL_GPUGraphicsPipeline *pipeline = nullptr;
};

static SDL_AppResult fail(const char *what) { SDL_Log("%s: %s", what, SDL_GetError()); return SDL_APP_FAILURE; }

SDL_AppResult SDL_AppInit(void **appstate, int, char *[])
{
    SDL_SetAppMetadata("Hello Triangle", "1.0", "com.example.sdl3.tri");
    if (!SDL_Init(SDL_INIT_VIDEO)) return fail("SDL_Init");

    AppState *st = new AppState(); *appstate = st;

    st->device = SDL_CreateGPUDevice(SDL_GPU_SHADERFORMAT_MSL, true, nullptr);
    if (!st->device) return fail("CreateGPUDevice");

    st->window = SDL_CreateWindow("Hello Triangle", 800, 600, SDL_WINDOW_RESIZABLE);
    if (!st->window) return fail("CreateWindow");

    if (!SDL_ClaimWindowForGPUDevice(st->device, st->window)) return fail("ClaimWindowForGPUDevice");

    SDL_GPUShaderCreateInfo vsi{};
    vsi.code = (const Uint8 *)kVS; vsi.code_size = std::strlen(kVS);
    vsi.entrypoint = "vertexMain";
    vsi.format = SDL_GPU_SHADERFORMAT_MSL; vsi.stage = SDL_GPU_SHADERSTAGE_VERTEX;
    SDL_GPUShader *vs = SDL_CreateGPUShader(st->device, &vsi);
    if (!vs) return fail("CreateGPUShader vs");

    SDL_GPUShaderCreateInfo fsi{};
    fsi.code = (const Uint8 *)kFS; fsi.code_size = std::strlen(kFS);
    fsi.entrypoint = "fragmentMain";
    fsi.format = SDL_GPU_SHADERFORMAT_MSL; fsi.stage = SDL_GPU_SHADERSTAGE_FRAGMENT;
    SDL_GPUShader *fs = SDL_CreateGPUShader(st->device, &fsi);
    if (!fs) { SDL_ReleaseGPUShader(st->device, vs); return fail("CreateGPUShader fs"); }

    SDL_GPUColorTargetDescription ct{};
    ct.format = SDL_GetGPUSwapchainTextureFormat(st->device, st->window);

    SDL_GPUGraphicsPipelineCreateInfo pci{};
    pci.vertex_shader = vs; pci.fragment_shader = fs;
    pci.primitive_type = SDL_GPU_PRIMITIVETYPE_TRIANGLELIST;
    pci.target_info.color_target_descriptions = &ct;
    pci.target_info.num_color_targets = 1;

    st->pipeline = SDL_CreateGPUGraphicsPipeline(st->device, &pci);
    SDL_ReleaseGPUShader(st->device, vs);
    SDL_ReleaseGPUShader(st->device, fs);
    if (!st->pipeline) return fail("CreateGPUGraphicsPipeline");

    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *, SDL_Event *e)
{
    if (e->type == SDL_EVENT_QUIT) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_KEY_DOWN && e->key.scancode == SDL_SCANCODE_ESCAPE) return SDL_APP_SUCCESS;
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    SDL_GPUCommandBuffer *cmd = SDL_AcquireGPUCommandBuffer(st->device);
    if (!cmd) return fail("AcquireGPUCommandBuffer");

    SDL_GPUTexture *swap = nullptr;
    Uint32 sw = 0, sh = 0;
    if (!SDL_WaitAndAcquireGPUSwapchainTexture(cmd, st->window, &swap, &sw, &sh))
        return fail("WaitAndAcquireGPUSwapchainTexture");

    if (swap) {
        SDL_GPUColorTargetInfo cti{};
        cti.texture     = swap;
        cti.clear_color = SDL_FColor{ 0.07f, 0.07f, 0.10f, 1.0f };
        cti.load_op     = SDL_GPU_LOADOP_CLEAR;
        cti.store_op    = SDL_GPU_STOREOP_STORE;
        SDL_GPURenderPass *pass = SDL_BeginGPURenderPass(cmd, &cti, 1, nullptr);
        SDL_BindGPUGraphicsPipeline(pass, st->pipeline);
        SDL_DrawGPUPrimitives(pass, 3, 1, 0, 0);
        SDL_EndGPURenderPass(pass);
    }
    SDL_SubmitGPUCommandBuffer(cmd);
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (!st) return;
    if (st->device) {
        SDL_ReleaseGPUGraphicsPipeline(st->device, st->pipeline);
        if (st->window) SDL_ReleaseWindowFromGPUDevice(st->device, st->window);
        SDL_DestroyGPUDevice(st->device);
    }
    SDL_DestroyWindow(st->window);
    delete st;
}

Try it yourself

  1. Change the three positions in the vertex shader to make a much smaller triangle. Recompile, observe — the shader source is rebuilt at runtime each launch.
  2. Change the three colours to all red. The triangle now goes flat-red because every interpolated value is red.
  3. Replace the fragment shader body with return float4(in.position.xy / 800.0, 0.0, 1.0); — the triangle becomes a screen-space gradient.
  4. Add a fourth corner to the position/color arrays and ask the draw call for 4 vertices. The fourth corner is ignored because PRIMITIVETYPE_TRIANGLELIST consumes 3 at a time. Switch to TRIANGLESTRIP and the fourth vertex becomes a second triangle.
