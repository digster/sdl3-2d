3D Track

Thirteen recipes that take you from the "hello triangle" to a textured, lit cube you can fly around with an FPS camera — and finally to cross-platform shaders authored once in HLSL and shipped to every backend. All in C++ using SDL3's SDL_GPU API (Metal / Vulkan / D3D12 under one abstraction).

Curriculum

  • 01 Hello Triangle
    GPU device, swapchain, shaders, pipeline, render pass — the smallest legal SDL_GPU program.
     Beginner
  • 02 Cross-Platform Shaders
    Why MSL is fine on macOS, why SPIR-V matters elsewhere, and how to ship both.
     Beginner
  • 03 Vertex Buffers
    Real geometry instead of [[vertex_id]] tricks — GPU buffers, transfer buffers, input state.
     Intermediate
  • 04 Index Buffers & Quads
    Drawing a quad with two triangles using 6 indices into 4 vertices.
     Intermediate
  • 05 Uniforms & the MVP Matrix
    Push constants, a hand-written 4×4 matrix, the model-view-projection pipeline.
     Intermediate
  • 06 The 3D Cube
    24 vertices, 36 indices, a rotating MVP. Your first real 3D object.
     Intermediate
  • 07 Depth Testing
    Why the cube looks wrong without it. Depth attachments and depth-stencil state.
     Intermediate
  • 08 Textures & Samplers
    Uploading a 2D image to the GPU, binding it to a fragment-shader sampler, UV coordinates.
     Advanced
  • 09 Phong Lighting
    Per-fragment lighting with a normal vertex attribute and a light-position uniform.
     Advanced
  • 10 Multiple Objects
    A small grid of cubes — one pipeline, one mesh, many SDL_DrawGPUIndexedPrimitives calls with different model matrices.
     Advanced
  • 11 FPS Camera
    Yaw/pitch from mouse motion, WASD relative to view, relative-mouse mode.
     Advanced
  • 12 Render-to-Texture
    Capstone — render the scene to an offscreen texture, then a fullscreen post-process pass.
     Advanced
  • 13 Shadercross in Practice
    Author shaders once in HLSL; ship one binary that runs on Metal, Vulkan and D3D12. Both the runtime and offline workflows, via SDL_shadercross.
     Advanced

How to run the code

For recipes 01–12, every Complete listing is a drop-in replacement for examples/cpp-gpu/triangle.cpp. Replace that file's contents, then build with make run-gpu from the repo root.

Recipe 13 lives in the sibling folder examples/cpp-gpu-shadercross/ because it has one extra dependency (SDL_shadercross), and is built with make run-shadercross.

Shaders in recipes 01–12 are written in Metal Shading Language (MSL) and embedded as raw string literals. On macOS that compiles at runtime and needs no extra tooling. On other platforms see recipe 02 for the SPIR-V / DXIL story, then jump to recipe 13 for the real cross-platform solution.
