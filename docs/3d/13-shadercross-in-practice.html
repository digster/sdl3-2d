Recipe 13 — Shadercross in Practice

Recipe 02 explained why cross-platform shaders are hard. This one solves the problem the way real engines do: author once in HLSL, cross-compile to whatever the device's backend natively wants, ship one binary that runs on Metal, Vulkan and D3D12.

What you'll learn

  • The four-function SDL_shadercross runtime API: Init, Quit, CompileSPIRVFromHLSL, CompileGraphicsShaderFromSPIRV.
  • Why SDL_ShaderCross_CompileGraphicsShaderFromSPIRV is the only call that has to know about the backend — it transparently transpiles SPIR-V to MSL/DXIL when the device asks for it.
  • The two delivery patterns: runtime (HLSL source compiled on launch) and offline (pre-compiled .spv loaded from disk via the shadercross CLI).
  • How HLSL semantics (SV_VertexID, SV_Position, SV_Target) are what makes one source portable.
  • When to pick which workflow, and why most shipped games do offline.

Prerequisites

Recipe 01 — Hello Triangle for the pipeline mechanics, and Recipe 02 — Cross-Platform Shaders for the conceptual map this recipe builds on. You'll also need SDL_shadercross and glslangValidator installed. shadercross isn't in Homebrew yet; the full macOS build recipe lives in examples/cpp-gpu-shadercross/README.md. The short version: 

brew install spirv-cross glslang
git clone --recursive https://github.com/libsdl-org/SDL_shadercross.git
cd SDL_shadercross
cmake -B build -DSDLSHADERCROSS_INSTALL=ON -DSDLSHADERCROSS_DXC=OFF
cmake --build build
sudo cmake --install build
Why both? shadercross can take HLSL or SPIR-V as input. HLSL → SPIR-V goes through DXC (not bundled on macOS); SPIR-V → MSL/DXIL goes through SPIRV-Cross (always bundled). So the cross-platform-dispatch part runs without DXC; the recipe just uses glslangValidator to produce the SPIR-V from GLSL — no DXC, no Microsoft compiler. The Advanced section of the sibling README explains how to add DXC if you want to author in HLSL.

Walkthrough

The same triangle, a different shader path

Side by side with examples/cpp-gpu/triangle.cpp, the only difference in this example is how the shaders enter the pipeline. The window, command buffer, render pass, draw call, and even the gradient triangle on screen are identical. Diff the two files; the deltas are entirely in SDL_AppInit around shader creation.

HLSL, with semantics

Author both stages in HLSL. The magic that makes the same source work on every backend is HLSL's semantics: SV_VertexID for the per-vertex index, SV_Position for the clip-space position output, SV_Target for the colour render target. DXC compiles these to SPIR-V with the right BuiltIn decorations; SPIRV-Cross then translates them into Metal's [[position]] / [[vertex_id]] / [[color(0)]] on macOS, or leaves them as Vulkan SPIR-V on Linux. Without semantics, cross-compilation would have to guess intent. 

// triangle.vert.hlsl  — three baked corners selected by SV_VertexID.
struct VSOut {
    float4 position : SV_Position;
    float4 color    : COLOR0;
};

VSOut main(uint vid : SV_VertexID)
{
    float2 positions[3] = {
        float2( 0.0f,  0.5f), float2(-0.5f, -0.5f), float2(0.5f, -0.5f)
    };
    float3 colors[3] = {
        float3(1,0,0), float3(0,1,0), float3(0,0,1)
    };
    VSOut o;
    o.position = float4(positions[vid], 0, 1);
    o.color    = float4(colors[vid], 1);
    return o;
}
// triangle.frag.hlsl — pass the interpolated colour through.
struct VSOut {
    float4 position : SV_Position;
    float4 color    : COLOR0;
};

float4 main(VSOut input) : SV_Target { return input.color; }

The runtime path: ~10 lines per stage

Two shadercross calls per stage. First, HLSL → SPIR-V: 

SDL_ShaderCross_HLSL_Info hi{};
hi.source       = kVertexShaderHLSL;
hi.entrypoint   = "main";
hi.shader_stage = SDL_SHADERCROSS_SHADERSTAGE_VERTEX;
size_t  spirv_size = 0;
void   *spirv      = SDL_ShaderCross_CompileSPIRVFromHLSL(&hi, &spirv_size);
if (!spirv) { /* SDL_GetError() */ }

Then SPIR-V → backend-native SDL_GPUShader. This second call is the one that actually knows about the backend — it transpiles to MSL on macOS, keeps SPIR-V on Vulkan, compiles to DXIL on D3D12, all without your code branching: 

SDL_ShaderCross_SPIRV_Info si{};
si.bytecode      = (const Uint8 *)spirv;
si.bytecode_size = spirv_size;
si.entrypoint    = "main";
si.shader_stage  = SDL_SHADERCROSS_SHADERSTAGE_VERTEX;

SDL_ShaderCross_GraphicsShaderResourceInfo ri{};   // all zero — no bindings

SDL_GPUShader *vs =
    SDL_ShaderCross_CompileGraphicsShaderFromSPIRV(device, &si, &ri, /*props=*/0);
SDL_free(spirv);   // shader has consumed the bytes now

The offline path: same second call, different source for the SPIR-V

Real games don't run a shader compiler on the player's machine. Instead, they compile SPIR-V at build time, ship the bytecode, and only call the second shadercross function (the one that produces the SDL_GPUShader) at runtime. The Makefile's shaders target uses glslangValidator on sibling GLSL sources because it's the no-DXC path: 

glslangValidator -V shaders/triangle.vert.glsl -o shaders/build/triangle.vert.spv
glslangValidator -V shaders/triangle.frag.glsl -o shaders/build/triangle.frag.spv

Authoring in HLSL instead? make shaders-hlsl runs the shadercross CLI itself, which can emit SPIR-V plus readable MSL / DXIL side-outputs you can inspect: 

shadercross triangle.vert.hlsl -s HLSL -d SPIRV -t vertex   -o triangle.vert.spv
shadercross triangle.vert.hlsl -s HLSL -d MSL   -t vertex   -o triangle.vert.msl
shadercross triangle.vert.hlsl -s HLSL -d DXIL  -t vertex   -o triangle.vert.dxil

Either way, at runtime you load the .spv and hand it to the same CompileGraphicsShaderFromSPIRV: 

size_t  spirv_size = 0;
void   *spirv = SDL_LoadFile("shaders/build/triangle.vert.spv", &spirv_size);
// ... same SDL_ShaderCross_SPIRV_Info as above, same call, same result.

That's the entire difference between the two modes — about ten lines. The example binary takes a --mode spirv|runtime flag (spirv is the default) so you can run both back-to-back from the same build.

The diagnostic log: see what the device actually picked

On startup, ask the device which formats it accepted, and log them. This is what tells you whether shadercross is going to do a transpile on this run, or hand the SPIR-V straight through: 

SDL_GPUShaderFormat f = SDL_GetGPUShaderFormats(device);
if (f & SDL_GPU_SHADERFORMAT_SPIRV)    SDL_Log("  - SPIRV    (Vulkan)");
if (f & SDL_GPU_SHADERFORMAT_MSL)      SDL_Log("  - MSL      (Metal source)");
if (f & SDL_GPU_SHADERFORMAT_METALLIB) SDL_Log("  - METALLIB (compiled Metal)");
if (f & SDL_GPU_SHADERFORMAT_DXIL)     SDL_Log("  - DXIL     (D3D12)");
if (f & SDL_GPU_SHADERFORMAT_DXBC)     SDL_Log("  - DXBC     (D3D11)");

On macOS expect MSL (and optionally METALLIB); on Linux expect SPIRV; on Windows expect DXIL. Crucially the rest of the code does not branch on this — it's purely informational.

Runtime vs offline — which to pick? Offline is what ships in 95% of games: faster cold-start (no DXC on the user's machine), smaller binary (no DXC linked in), easier symbol-level debugging of the SPIR-V. Runtime is the right call for editors, shader hot-reload, procedurally generated shaders, or learning — when the convenience of "edit a string, rerun" outweighs the cold-start cost.

Complete listing

The full file from the repo, around 340 lines. Drop into examples/cpp-gpu-shadercross/triangle_shadercross.cpp and build with make run-shadercross (after installing shadercross per the prerequisites above). 

/*
 * triangle_shadercross.cpp — SDL3 GPU + SDL_shadercross cross-platform shaders.
 * See the file's header comment for the full design notes. Abridged below to
 * just the moving parts.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>
#include <SDL3_shadercross/SDL_shadercross.h>

#include <cstring>
#include <cstdlib>
#include <string>

static const char *kVertexShaderHLSL = R"hlsl(
struct VSOut { float4 position : SV_Position; float4 color : COLOR0; };
VSOut main(uint vid : SV_VertexID) {
    float2 P[3] = { float2(0,0.5), float2(-0.5,-0.5), float2(0.5,-0.5) };
    float3 C[3] = { float3(1,0,0), float3(0,1,0), float3(0,0,1) };
    VSOut o; o.position = float4(P[vid],0,1); o.color = float4(C[vid],1); return o;
}
)hlsl";

static const char *kFragmentShaderHLSL = R"hlsl(
struct VSOut { float4 position : SV_Position; float4 color : COLOR0; };
float4 main(VSOut input) : SV_Target { return input.color; }
)hlsl";

enum class ShaderMode { RUNTIME, SPIRV };

struct AppState {
    SDL_Window              *window   = nullptr;
    SDL_GPUDevice           *device   = nullptr;
    SDL_GPUGraphicsPipeline *pipeline = nullptr;
    long       max_frames = -1;
    long       frame      = 0;
    ShaderMode mode       = ShaderMode::RUNTIME;
};

static SDL_AppResult fail(const char *what) {
    SDL_Log("%s: %s", what, SDL_GetError());
    return SDL_APP_FAILURE;
}

static void log_device_shader_formats(SDL_GPUShaderFormat f) {
    SDL_Log("device-accepted shader formats:");
    if (f & SDL_GPU_SHADERFORMAT_SPIRV)    SDL_Log("  - SPIRV    (Vulkan)");
    if (f & SDL_GPU_SHADERFORMAT_MSL)      SDL_Log("  - MSL      (Metal source)");
    if (f & SDL_GPU_SHADERFORMAT_METALLIB) SDL_Log("  - METALLIB (compiled Metal)");
    if (f & SDL_GPU_SHADERFORMAT_DXIL)     SDL_Log("  - DXIL     (D3D12)");
    if (f & SDL_GPU_SHADERFORMAT_DXBC)     SDL_Log("  - DXBC     (D3D11)");
}

/* Mode-agnostic: compile an HLSL string or load a .spv, then produce a shader. */
static SDL_GPUShader *load_stage(SDL_GPUDevice *device, ShaderMode mode,
                                 const char *hlsl, const char *spv_basename,
                                 SDL_ShaderCross_ShaderStage stage)
{
    size_t  size = 0;
    void   *bytes = nullptr;

    if (mode == ShaderMode::RUNTIME) {
        SDL_ShaderCross_HLSL_Info hi{};
        hi.source = hlsl; hi.entrypoint = "main"; hi.shader_stage = stage;
        bytes = SDL_ShaderCross_CompileSPIRVFromHLSL(&hi, &size);
    } else {
        std::string path = std::string("examples/cpp-gpu-shadercross/shaders/build/") + spv_basename;
        bytes = SDL_LoadFile(path.c_str(), &size);
        if (!bytes) SDL_Log("did you run `make shaders`? %s", SDL_GetError());
    }
    if (!bytes) return nullptr;

    SDL_ShaderCross_SPIRV_Info si{};
    si.bytecode = (const Uint8 *)bytes; si.bytecode_size = size;
    si.entrypoint = "main"; si.shader_stage = stage;

    SDL_ShaderCross_GraphicsShaderResourceInfo ri{};   // all zero — no bindings
    SDL_GPUShader *sh =
        SDL_ShaderCross_CompileGraphicsShaderFromSPIRV(device, &si, &ri, 0);
    SDL_free(bytes);
    return sh;
}

SDL_AppResult SDL_AppInit(void **appstate, int argc, char *argv[]) {
    if (!SDL_Init(SDL_INIT_VIDEO)) return fail("SDL_Init");
    AppState *st = new AppState();
    *appstate = st;

    for (int i = 1; i < argc; ++i) {
        if (!std::strcmp(argv[i], "--frames") && i + 1 < argc)
            st->max_frames = std::atol(argv[++i]);
        else if (!std::strcmp(argv[i], "--mode") && i + 1 < argc) {
            const char *m = argv[++i];
            if      (!std::strcmp(m, "runtime")) st->mode = ShaderMode::RUNTIME;
            else if (!std::strcmp(m, "spirv"))   st->mode = ShaderMode::SPIRV;
        }
    }

    if (!SDL_ShaderCross_Init()) return fail("SDL_ShaderCross_Init");

    // Advertise every format — SDL picks what the actual backend wants.
    SDL_GPUShaderFormat fmts = SDL_GPU_SHADERFORMAT_SPIRV | SDL_GPU_SHADERFORMAT_MSL |
                               SDL_GPU_SHADERFORMAT_METALLIB | SDL_GPU_SHADERFORMAT_DXIL |
                               SDL_GPU_SHADERFORMAT_DXBC;
    st->device = SDL_CreateGPUDevice(fmts, /*debug=*/true, nullptr);
    if (!st->device) return fail("SDL_CreateGPUDevice");
    log_device_shader_formats(SDL_GetGPUShaderFormats(st->device));

    st->window = SDL_CreateWindow("SDL3 GPU + shadercross", 800, 600, SDL_WINDOW_RESIZABLE);
    SDL_ClaimWindowForGPUDevice(st->device, st->window);

    SDL_GPUShader *vs = load_stage(st->device, st->mode, kVertexShaderHLSL,
                                   "triangle.vert.spv",
                                   SDL_SHADERCROSS_SHADERSTAGE_VERTEX);
    SDL_GPUShader *fs = load_stage(st->device, st->mode, kFragmentShaderHLSL,
                                   "triangle.frag.spv",
                                   SDL_SHADERCROSS_SHADERSTAGE_FRAGMENT);
    if (!vs || !fs) return SDL_APP_FAILURE;

    SDL_GPUColorTargetDescription ct{};
    ct.format = SDL_GetGPUSwapchainTextureFormat(st->device, st->window);
    SDL_GPUGraphicsPipelineCreateInfo pci{};
    pci.vertex_shader = vs; pci.fragment_shader = fs;
    pci.primitive_type = SDL_GPU_PRIMITIVETYPE_TRIANGLELIST;
    pci.target_info.color_target_descriptions = &ct;
    pci.target_info.num_color_targets = 1;
    st->pipeline = SDL_CreateGPUGraphicsPipeline(st->device, &pci);

    SDL_ReleaseGPUShader(st->device, vs);
    SDL_ReleaseGPUShader(st->device, fs);
    if (!st->pipeline) return fail("SDL_CreateGPUGraphicsPipeline");
    return SDL_APP_CONTINUE;
}

/* SDL_AppEvent / SDL_AppIterate are identical to recipe 01 — see that recipe
 * for the per-frame loop with the NULL-swapchain handling. SDL_AppQuit adds
 * one line: SDL_ShaderCross_Quit() to balance the Init above. */

Try it yourself

  1. Run the example in the default (spirv) mode and read the startup log: 
    make shaders && make run-shadercross
    You should see "shader mode: spirv (loaded from .spv on disk…)" then a list of the formats your device accepted — on macOS that's MSL (and probably METALLIB). The triangle renders the same as the embedded-MSL example in examples/cpp-gpu/.
  2. Edit one of the corner colours in examples/cpp-gpu-shadercross/shaders/triangle.vert.glsl, re-run make shaders && make run-shadercross, and see the change. The SPIR-V is regenerated by glslangValidator; the C++ doesn't need recompiling.
  3. Built shadercross with DXC? Try make shaders-hlsl to cross-compile the HLSL sources instead — alongside .spv you'll see .msl and (if DXC is enabled) .dxil files appear in shaders/build/. Open the .msl files in an editor — that's the actual Metal shader your GPU will run. Compare it line by line with the embedded MSL in examples/cpp-gpu/triangle.cpp.
  4. Also with DXC, exercise the runtime path: ./build/demo_gpu_shadercross --mode runtime. The startup log will read "shader mode: runtime …" and the embedded HLSL string gets compiled in-process. Without DXC the example logs a clear hint and points back to this page.
  5. (Advanced) Add a uniform buffer — say, a single float for an animated colour shift. Bump num_uniform_buffers in the SDL_ShaderCross_GraphicsShaderResourceInfo you pass in, or replace the hand-counted struct with a SDL_ShaderCross_ReflectGraphicsSPIRV call that fills it from the SPIR-V itself.
