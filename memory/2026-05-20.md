# 2026-05-20 — fourth example: SDL_GPU + SDL_shadercross

Added a fourth, fully independent example: `examples/cpp-gpu-shadercross/`,
the cross-platform shaders demo using SDL's official cross-compiler library
[SDL_shadercross](https://github.com/libsdl-org/SDL_shadercross). Same triangle
as `cpp-gpu/triangle.cpp`, but shaders authored in HLSL/GLSL and dispatched to
whatever the device's GPU backend natively wants (MSL on macOS, SPIR-V on
Vulkan, DXIL on D3D12).

## Files added

- `examples/cpp-gpu-shadercross/triangle_shadercross.cpp` — main example,
  ~360 lines, mirrors `cpp-gpu/triangle.cpp` callback-loop shape so the
  diff between the two files is purely the shader-acquisition path.
- `examples/cpp-gpu-shadercross/shaders/triangle.{vert,frag}.hlsl` — HLSL
  sources, also embedded as raw strings in the C++ for `--mode runtime`.
- `examples/cpp-gpu-shadercross/shaders/triangle.{vert,frag}.glsl` — GLSL
  twins of the above; the no-DXC path uses these via `glslangValidator`.
- `examples/cpp-gpu-shadercross/README.md` — install steps for
  `SDL_shadercross` on macOS, dual-toolchain explanation, troubleshooting.
- `docs/3d/13-shadercross-in-practice.html` — new advanced recipe (added
  as #13 rather than re-numbering 02-12 to keep existing footer-nav links
  intact). Cross-linked from recipe 02 (see-also callout) and recipe 12
  (next-link).

## Files edited

## Key decisions

1. **Independent example folder, no shared source.** Honours the
   ARCHITECTURE.md independence rule. Diffing `triangle_shadercross.cpp`
   against `cpp-gpu/triangle.cpp` shows the cross-platform delta in
   isolation; the body of the file is intentionally near-identical.

2. **Optional dependency, not a hard prerequisite.** The repo's
   "just `brew install sdl3`" promise stays intact for the three existing
   examples. shadercross is `pkg-config --exists`-detected by both build
   systems; if missing, the other three demos build cleanly and a friendly
   note points at the sibling README's install steps. **Do not** promote
   shadercross to a hard prerequisite — the ARCHITECTURE.md convention
   list now documents this.

3. **GLSL+glslang as the primary `make shaders` path, not HLSL+shadercross-CLI.**
   Discovered during verification: `SDL_ShaderCross_CompileSPIRVFromHLSL`
   (and the `shadercross` CLI's HLSL pipeline) require DXC linked into
   shadercross, which is **not** bundled on macOS. The HLSL → SPIR-V step
   is the only thing that needs DXC; the cross-platform-dispatch piece
   (`SDL_ShaderCross_CompileGraphicsShaderFromSPIRV`) uses SPIRV-Cross and
   always works. So `make shaders` uses `glslangValidator` (Homebrew), and
   `make shaders-hlsl` is the DXC-required alternative. C++ default
   `--mode` is `spirv` (always-works); `--mode runtime` is the embedded-HLSL
   path with a helpful "rebuild shadercross with DXC" hint on failure.

4. **Recipe 13, not renumbering.** Adding the new recipe after the existing
   12 keeps every existing footer-nav link in recipes 01-12 valid. Recipe
   02 still introduces the *concept* of cross-platform shaders; recipe 13
   is the *production* solution. Recipe 02 gets a small "See also"
   callout pointing forward.

5. **Bake shadercross rpath at link time** via
   `-Wl,-rpath,$(pkg-config --variable=libdir sdl3-shadercross)`. macOS SIP
   strips `DYLD_*` env vars when `/bin/sh` (and therefore `make`) spawns
   binaries, so the runtime path can't be patched from the outside.

## Verification done

- `make clean && make` — clean build, four binaries produced. Build is
  warning-free except for two cosmetic linker dedup warnings (duplicate
  `-rpath` / `-lSDL3` coming from shadercross pkg-config not knowing what
  SDL3's pkg-config already added — harmless).
- `make shaders` — produces `triangle.{vert,frag}.spv` via glslangValidator.
- `make smoke` — all four demos pass `--frames 120` headless, exit 0.
- `./build/demo_gpu_shadercross --frames 30` (default `--mode spirv`) —
  exit 0, correct mode-and-device-formats log.
- `./build/demo_gpu_shadercross --mode runtime --frames 30` — exit 1 with
  the targeted "shadercross was not built with DXC, see README" hint (this
  is correct behaviour on this machine since the local shadercross install
  lacks DXC).

## Edge cases noted for future test coverage

- `make shaders-hlsl` end-to-end — needs a machine with DXC installed.
- Behaviour on a machine where shadercross's CMake config exports
  `SDL3_shadercross::SDL3_shadercross` rather than the `-shared`/`-static`
  variants — the CMakeLists.txt already has a three-way `if(TARGET …)`
  fallback chain but it's untested.
- pkg-config file name as `SDL3_shadercross` (capital S) rather than
  `sdl3-shadercross` — the Makefile probes both, untested.
- Live-reload scenario: edit a .glsl, re-run `make shaders`, restart the
  binary — should "just work" but worth a manual run.
- Adding a uniform buffer requires bumping `num_uniform_buffers` in the
  `SDL_ShaderCross_GraphicsShaderResourceInfo` — or using
  `SDL_ShaderCross_ReflectGraphicsSPIRV` to discover counts automatically.
  Test once we have a recipe extending this with real uniforms.
