# 2026-05-17 — SDL3 2D starter template bootstrapped

## What was done

Created a greenfield SDL3 2D starter for macOS from an empty directory.

- `include/gfx.h` — `extern "C"` drawing API (also the API reference). All
  float coords, "set colour then draw" convention.
- `src/gfx.c` — implementations: point, line, rect/fill_rect, circle (midpoint
  algorithm), fill_circle (scanline), triangle, fill_triangle
  (SDL_RenderGeometry; converts 0–255 colour to 0–1 SDL_FColor).
- `examples/traditional.c` — C, hand-written `while` game loop.
- `examples/callbacks.cpp` — C++, SDL3 `SDL_MAIN_USE_CALLBACKS` model with a
  heap `AppState` and a `std::vector` feeding the C API.
- `Makefile` — primary driver: `make`, `run-c`, `run-cpp`, `smoke`,
  `debug-c/-cpp` (lldb), `clean`; pkg-config based; `DEBUG=1` toggle;
  `check-sdl` guard.
- `CMakeLists.txt` — alternative path via `find_package(SDL3 CONFIG)`.
- Housekeeping: README.md, ARCHITECTURE.md, LICENSE (MIT/digster/2026),
  empty CLAUDE.md, PROMPT.md, .gitignore, this memory file.

Both examples accept `--frames N` for non-interactive smoke runs and draw the
same scene (every helper + a dt-driven bouncing ball).

## Key decisions / context

- One C example + one C++ example covers all four axes (C, C++, traditional,
  callbacks) with no duplicated source — the shared C `gfx` object links into
  both because `gfx.h` is `extern "C"`.
- SDL3 (not SDL2): `bool` returns where `true`=success; float render coords;
  callback runtime calls `SDL_Quit()` itself after `SDL_AppQuit`.
- Window created without high-DPI flag so render coords == window points 1:1
  (keeps examples simple).
